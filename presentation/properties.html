CSS Keyframe Animations

Delays

You can add a delay to the start of the animation. This is done by adding the property name "animation-delay" to the element's selector. Negative values can be entered and will cause the animation to start as if it had already been active for x number of seconds.

Example

div {
 animation-name: example;
 animation-duration: 4s;
 animation-delay: 2s;
}
@keyframes example {
 0% {background-color: #153e5c;}
 100% {background-color: #c1cdd5;}
}

In this example we are changing the background color from a darker blue (#153e5c) to a lighter blue (#c1cdd5) with a 2 second delay. Click the box below to watch this happen.

Example

div {
 animation-name: example;

 animation-delay: -2s;

In this example we are again changing the background color from a darker blue (#153e5c) to a lighter blue (#c1cdd5) but this time with a -2 seond delay. Click the box below to watch this happen.

Repeat

You can repeat the animation's cycle. This is done by adding the property name "animation-iteration-count" to the element's selector. To have the animination run an infinate number of times declare the value as "infinite."

Example

div {
 animation-name: example;
 animation-duration: 4s;
 animation-iteration-count: 3;
}
@keyframes example {
 0% {background-color: #153e5c;}
 100% {background-color: #c1cdd5;}
}

In this example we are changing the background color from a darker blue (#153e5c) to a lighter blue (#c1cdd5), and cycling through that sequence three times. Click the box below to watch this happen.

Example

div {
 animation-name: example;
 animation-duration: 4s;
 animation-iteration-count: infinite;
}
@keyframes example {
 0% {background-color: #153e5c;}
 100% {background-color: #c1cdd5;}
}

In this example we are changing the background color from a darker blue (#153e5c) to a lighter blue (#c1cdd5), and cycling through that sequence forever. Click the box below to watch this happen.

Direction

By default, the animation will play through once from beginning to end. However, we can alter this by adding the property name "animation-direction" to the element's selector. The animation-direction property can have the following values:

  • Normal: The animation plays through as normal, from beginning to end. This is the default.
  • Reverse: The animation plays through in reverse or backwards, from end to beginning.
  • Alternate: The animation plays through from beginning to end, then from end to beginning.
  • Alternate-reverse: The animation plays through from end to beginning, then from beginning to end.

Example

div {
 animation-name: example;
 animation-duration: 4s;
 animation-direction: normal;
}
@keyframes example {
 0% {background-color:#153e5c; left:0px; top:0px;}
 25% {background-color:#ffffff; left:100px; top:0px;}
 50% {background-color:#698497; left:100px; top:100px;}
 75% {background-color:#000000; left:0px; top:100px;}
 100% {background-color:#153e5c; left:0px; top:0px;}
}

In this example we are changing the background color from a darker blue in the upper left corner (at 0%) to white in the upper right corner (at 25%) to a lighter blue in the bottom right corner (at 50%) to black in the bottom left corner (at 75%) and then back to the dark blue we started with in the upper left corner (at 100%), and cycling through the sequence in normal. Click the box below to watch this happen.

Timing

You can specify the timing of the animation by using the animation-timing-function property. By default, the animation will follow the ease value, meaning that it will start and end the animation slowly, and have a fast middle. The animation-timing-function property can have the following values:

  • Ease: Animation will start and end slowly, and have a fast middle. This is the default.
  • Linear: Animation will have the same speed from start to end.
  • Ease-in: Animation will start slowly.
  • Ease-out: Animation will end slowly.
  • Ease-in-out: Animation will start and end slowly.
  • Cubic-bezier(P0, P1, P2, P3): Allows you to define your own values in a cubic-bezier function.

The Cubic-bezier curve is defined by four points. These points are generally referred in one of the following ways. First as P0, P1, P2, and P3 as (P0, P1, P2, P3); or x1, y1, x2, and y2 as (x1, y1, x2, y2). All four points are required in the code as numeric values. Additionally, P0/x1 and P2/x2 must be a number from 0 to 1.

Example

div {
 animation-name: example;
 animation-duration: 10s;
 animation-iteration-count: 3;
 animation-timing-function: value;
}
@keyframes example {
 0% {left:0px;}
 100% {left:750px;}
}

In this example we are moving the boxes from one side of the screen to the other using each of the animation-timing-function properties available. Our cubic-bezier will have the following values: (0, 1, 0.5, 0.2). This is repeated three times. Click any of the boxes below to watch this happen.






Fill

Generally, the animations will not affect the elements properties before or after the animation is played. You can override this using the animation-fill-mode property. By default, the animation will follow the none value, meaning that the animation will only affect the element during the animation play. Please keep in mind that adding an animation-direction value may alter the effect of the animation-fill-mode. The animation-fill-mode property can have the following values:

  • None: The element will not keep the styles from the animation. This is the default.
  • Forwards: The element will keep the values set by the last keyframe after the animation is played.
  • Backwards: The element will have the values set by the first keyframe, before the animation is played.
  • Both: The element will have the values set by the first keyframe, before the animation is played, and keep the values set by the last keyframe after the animation is played.

Example

div {
 background-color: #153e5c;
 animation-name: example;
 animation-duration: 4s;
 animation-delay: 2s;
 animation-fill-mode: value;
}
@keyframes example {
 0% {background-color: #000000; left:0px;}
 100% {background-color: #698497; left:750px;}
}

In this example we are moving the boxes from one side of the screen to the other and changing their color from black to light blue, using each of the animation-fill-mode properties available. So that the animation-fill-mode options can be compared accurately a 2 second animation delay has also been added. Click any of the boxes below to watch this happen.




Shorthand

To illustrate the use of the shorthand animation property the following two examples create the same animation effect which can be seen in action by clicking on the box below.

Example

div {
 animation-name: example;
 animation-duration: 4s;
 animation-timing-function: ease-in;
 animation-delay: 2s;
 animation-iteration-count: infinite;
 animation-fill-mode: none;
}

Example

div {
 animation: example 4s ease-in 2s infinite none;
}

This example uses all of the values specified above, but adds some new styles in the @keyframes that we haven't used yet. Click the box below to see what happens.
